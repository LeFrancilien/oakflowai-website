n8n Cloud : tarifs 2026, limites et quand l'utiliser

n8n Cloud, c'est la version hébergée et sans maintenance de n8n. Pratique pour démarrer, mais facturée à l'exécution et plafonnée. Voici les tarifs réels, les limites à connaître, et les critères concrets pour choisir entre le cloud et l'auto-hébergement.

n8n Cloud, c'est quoi exactement ?

n8n Cloud est l'offre hébergée par l'éditeur de n8n. Vous n'installez rien : vous ouvrez un compte, et vos workflows tournent sur l'infrastructure de n8n, avec les mises à jour et la maintenance gérées pour vous. C'est l'inverse du self-hosted, où vous déployez n8n sur votre propre serveur.

Point clé à comprendre avant de parler prix : n8n facture à l'exécution, pas à l'opération. Un lancement complet d'un workflow compte pour une seule exécution, quel que soit le nombre d'étapes (10 ou 100) ou le volume de données traité. C'est un avantage structurel face à Zapier (facturation à la tâche) et Make (facturation à l'opération), détaillé dans notre comparatif n8n vs Zapier vs Make.

Voici les plans n8n Cloud (tarifs en facturation annuelle, au moment de la rédaction). Les prix évoluent : vérifiez toujours la grille à jour sur n8n.io avant de vous engager.

Le nombre d'workflows actifs est illimité sur tous les plans : ce qui compte, c'est le total d'exécutions mensuelles. C'est donc le volume, pas le nombre d'automatisations, qui détermine votre facture.

Avantages et limites de n8n Cloud

Ce que le cloud vous apporte

  • Zéro maintenance : pas de serveur à gérer, mises à jour et sauvegardes incluses.
  • Démarrage immédiat : un compte, et vous construisez vos premiers workflows en minutes.
  • Support éditeur : assistance officielle selon votre plan.

Les limites à connaître

  • Plafond d'exécutions : au-delà du quota, il faut passer au plan supérieur. Sur des volumes élevés, le coût grimpe vite face à un self-hosted.
  • Moins de contrôle : vos données transitent par l'infrastructure de l'éditeur, et vous dépendez de sa disponibilité.
  • Coût au volume : un workflow déclenché très fréquemment (webhooks, polling) peut consommer beaucoup d'exécutions.

Sécurité des données et RGPD

n8n Cloud propose un hébergement des données dans l'Union européenne, avec des communications chiffrées. Pour la majorité des usages PME, c'est conforme au RGPD. Mais si vous traitez des données particulièrement sensibles (santé, RH, juridique) et que vous voulez la garantie que rien ne quitte votre propre infrastructure, le self-hosted sur serveur français reste la configuration la plus stricte. Nous détaillons les deux dans notre guide d'hébergement d'un serveur n8n.

Cloud ou self-hosted : comment décider ?

Votre situation Recommandation
Vous testez, petits volumes, pas d'équipe technique n8n Cloud (Starter/Pro)
Volumes d'exécutions élevés et croissants Self-hosted (VPS ~10-20 €/mois, exécutions illimitées)
Données sensibles, souveraineté critique Self-hosted sur serveur français

La bascule cloud → self-hosted est fréquente : on démarre sur le Cloud pour valider, puis on migre quand les volumes montent. Questions fréquentes sur n8n Cloud

Combien coûte n8n Cloud ?

n8n Cloud démarre à 20 €/mois (Starter, 2 500 exécutions), puis 50 €/mois (Pro, 10 000 exécutions) et 667 €/mois (Business, 40 000 exécutions), en facturation annuelle. Un plan Enterprise sur devis existe pour les gros volumes. La facturation se fait à l'exécution de workflow, quel que soit le nombre d'étapes.

n8n Cloud facture-t-il à l'opération comme Zapier ?

Non. n8n facture à l'exécution : un lancement complet d'un workflow compte pour une seule exécution, peu importe le nombre d'étapes ou le volume de données. C'est un avantage majeur face à Zapier (à la tâche) et Make (à l'opération), surtout pour les workflows à nombreuses étapes.

n8n Cloud est-il conforme au RGPD ?

n8n Cloud propose un hébergement des données dans l'Union européenne, avec chiffrement des communications. C'est conforme au RGPD pour la plupart des usages PME. Pour une souveraineté totale, le self-hosted sur serveur français reste la solution la plus stricte.

Faut-il choisir n8n Cloud ou le self-hosted ?

n8n Cloud convient pour démarrer vite, sans maintenance, sur des volumes maîtrisés. Le self-hosted devient plus avantageux dès que les volumes montent (exécutions illimitées pour le prix d'un VPS à 10-20 €/mois) ou quand la souveraineté des données est critique. Beaucoup de PME commencent sur le Cloud puis basculent.

Y a-t-il un plan gratuit sur n8n Cloud ?

n8n Cloud propose un essai gratuit, mais pas de plan gratuit permanent. La seule version réellement gratuite de n8n est la version self-hosted (open source), que vous installez sur votre propre serveur : vous ne payez alors que l'hébergement.

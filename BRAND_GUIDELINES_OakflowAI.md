## 3. Identité Visuelle

### Concept Logo

- **Oak** = Robustesse, croissance, ancrage (chêne)
- **Flow** = Fluidité, technologie, flux de données
- Arbre stylisé + nœuds de données interconnectés
- Style : **minimaliste, tech, organique**

### Palette de Couleurs

| Rôle | Couleur | Usage |
|---|---|---|
| **Fond principal** | `#0a0f0e` (noir profond vert-noir) | Backgrounds sections |
| **Fond secondaire** | `#0d1210` | Cartes, nav |
| **Accent primaire** | `#00e5b0` (cyan/teal néon) | CTA, highlights, liens actifs |
| **Accent secondaire** | `#00c49a` (teal légèrement plus sombre) | Hover states, bordures actives |
| **Texte principal** | `#ffffff` | Titres, corps |
| **Texte secondaire** | `#a0b0ad` | Sous-titres, descriptions |
| **Succès / Check** | `#00e5b0` | Icônes de validation ✅ |

> **Mode :** Dark mode exclusif. Pas de version light mode.

### Typographie

| Usage | Police | Style |
|---|---|---|
| **Titres** | `Inter` ou `Geist` | Bold / ExtraBold |
| **Corps** | `Inter` ou `Geist` | Regular / Medium |
| **Code / Technique** | `Geist Mono` | Regular |

> Priorité : **Geist** (Vercel) pour l'aspect tech premium. **Inter** en fallback.

### Style Visuel

- **Dark mode néon/organique** — fonds très sombres, accents néon turquoise
- Éléments **3D interactifs** (Spline) : nœuds de données qui se connectent
- Grille **Bento Box** pour le portfolio
- Animations fluides (scroll, mouse tracking, transitions)
- Icônes minimalistes (outline ou fill léger)

### CTA Primaire (Bouton)

- Fond : `#00e5b0` (teal néon)
- Texte : `#000000` ou `#0a0f0e` (noir)
- Border-radius : `8px` — style arrondi mais rectangulaire
- Exemples : `"Planifier un appel découverte gratuit (30 min)"`, `"Réserver 30min"`, `"Voir les cas d'usage"`

### CTA Secondaire (Bouton Ghost)

- Fond : transparent
- Bordure : `1px solid #ffffff` ou `#00e5b0`
- Texte : `#ffffff`
- Exemple : `"Découvrir nos Solutions"`

## 5. Tone of Voice

### Principes

- **Professionnel mais accessible** — pas de jargon inutile
- **Orienté résultats** — toujours mettre en avant ROI, métriques, gains concrets
- **Confiant** — position d'expert, pas de formulations hésitantes
- **Direct** — phrases courtes, CTA clairs

### Formulations à privilégier

- ✅ *"30h économisées par mois"*, *"0 erreur de saisie"*
- ✅ *"Sur-mesure"*, *"Concret"*, *"Opérationnel"*
- ✅ *"Votre consultant"* (relation de proximité)
- ✅ *"Sans engagement"*, *"Diagnostic personnalisé"*, *"Solutions concrètes"*

### Formulations à éviter

- ❌ *"Révolutionnaire"*, *"Disruptif"* (trop générique)
- ❌ Jargon technique pur sans explication
- ❌ Formulations vagues sans métriques

## 7. Structure d'un Cas d'Usage (Use Case)

Chaque workflow présenté suit cette structure stricte :

1. **Problématique** — Contexte client initial *(ex: tri de 500 factures/mois)*
2. **Solution Technique** — Architecture n8n + LLM utilisé
3. **Résultats / ROI** — Métriques concrètes *(ex: 30h gagnées/mois, 0 erreur de saisie)*
4. **Visuel** — Capture stylisée ou animation du workflow n8n
